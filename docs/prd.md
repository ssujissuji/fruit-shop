# PRD — 과일 쇼핑몰 (fruit-shop)

**버전:** 1.2  
**작성일:** 2026-05-13  
**최종 수정일:** 2026-05-15  
**상태:** 찜하기 기능 구현 완료

---

## 1. 프로젝트 개요

### 목적
신선한 과일을 온라인으로 간편하게 구매할 수 있는 쇼핑몰 웹 애플리케이션을 구축한다.

### 목표 사용자
- 마트 방문 없이 집에서 신선한 과일을 주문하고 싶은 일반 소비자
- 모바일/PC 모두 사용 가능한 환경을 원하는 사용자

### 범위
- 상품 탐색 (홈 화면)
- 상품 상세 조회
- 장바구니 관리
- 회원가입 / 로그인 (v1.1 추가, 서버 없는 연습용 mock 구현)
- 마이페이지 (프로필 조회 + 이름 수정)
- 상품 검색 (홈 화면 검색창, 카테고리 배지 연동)
- 찜하기 (하트 버튼, 찜 목록 페이지, localStorage 영속화)
- (제외) 실제 결제, 주문 내역

## 2. 화면 목록

| 화면 | URL | 설명 |
|---|---|---|
| 홈 | `/` | 배너 슬라이더 + 전체 상품 목록 |
| 상품 상세 | `/product/:id` | 과일 상세 정보 + 구매 액션 |

| 회원가입 | `/signup` | 이름 + 이메일 + 비밀번호 가입 |

## 3. 화면별 기능 명세

### 3-1. 홈 화면 (`/`)

#### 배너 슬라이더
- 3개 이상의 프로모션 배너 이미지를 자동 슬라이드한다
- 좌/우 이동 버튼으로 수동 전환이 가능하다
- 현재 슬라이드 위치를 인디케이터(점)로 표시한다
- 자동 슬라이드 간격: 3초

#### 상품 목록
- 전체 과일 상품을 카드 형태로 나열한다
- 카드 1개에 표시되는 정보: 상품 이미지, 상품명, 가격
- 카드 클릭 시 해당 상품의 상세 페이지로 이동한다
- 레이아웃: 모바일 2열 / 태블릿 3열 / 데스크탑 4열 (반응형)

#### 상품 검색
- 상품명(한글/영문)을 입력해 실시간으로 상품 목록을 필터링한다
- 대소문자 구분 없이 검색된다
- 카테고리 필터와 AND 조건으로 함께 적용된다
- 카테고리가 선택된 상태에서 검색 시 입력창 왼쪽에 카테고리 배지가 표시된다
- 배지 클릭 시 카테고리가 전체로 해제된다
- 검색 결과가 없을 때 "검색 결과가 없습니다." 안내 문구를 표시한다

---

### 3-2. 상품 상세 페이지 (`/product/:id`)

#### 표시 정보
| 항목 | 내용 |
|---|---|
| 상품 이미지 | 대표 이미지 1장 |
| 상품명 | 한글명 + 영문명 |
| 가격 | 원화 형식 (예: 3,000원) |
| 원산지 | 텍스트 |
| 중량 | 텍스트 |
| 상품 설명 | 상세 텍스트 |
| 재고 | 재고 수량 표시 |

#### 구매 액션
- **수량 선택:** `-` / `+` 버튼으로 수량을 조절한다
  - 최소 수량: 1 (1 미만으로 감소 불가)
  - 최대 수량: 재고 수량 이하
- **장바구니 담기:** 선택 수량만큼 장바구니에 추가한다
  - 이미 담긴 상품이면 수량을 합산한다
- **구매하기:** 장바구니에 추가 후 장바구니 페이지로 즉시 이동한다
- 존재하지 않는 id 접근 시 홈으로 리다이렉트한다

---

### 3-3. 장바구니 페이지 (`/cart`)

#### 상품 목록
- 담긴 상품별로 행을 구성한다
- 행 표시 항목: 상품 이미지, 상품명, 단가, 수량, 소계 (단가 × 수량)

#### 수량 관리
- 각 상품의 `-` / `+` 버튼으로 수량을 조절한다
  - 최소 수량: 1
- **삭제 버튼:** 해당 상품을 장바구니에서 제거한다

#### 주문 요약
- 전체 합계 금액을 표시한다 (담긴 상품 소계의 합)
- **주문하기 버튼:** v1에서는 UI만 구현 (실제 결제 기능 제외)

#### 빈 장바구니 상태
- 담긴 상품이 없을 때 안내 문구와 홈으로 이동 버튼을 표시한다

---

### 3-4. 로그인 페이지 (`/login`)

- Header / Footer 없는 풀페이지 레이아웃
- 상단에 로고 바 표시
- **입력 필드:** 이메일, 비밀번호
- **유효성 검사 (submit 시):**
  - 빈 필드 → 필드 하단 인라인 에러 표시
  - 이메일/비밀번호 불일치 → 상단 에러 메시지 표시
- 로그인 성공 시 홈(`/`)으로 자동 이동
- 이미 로그인 상태에서 접근 시 홈으로 리다이렉트
- 하단에 회원가입 페이지 링크 제공
- 연습용 mock 계정 힌트 표시 (`test@fruit.com` / `1234`)

---

### 3-6. 마이페이지 (`/mypage`)

- Header / Footer 포함 (기존 App 레이아웃)
- 비로그인 상태에서 접근 시 `/login`으로 리다이렉트 (PrivateRoute)

#### 프로필 카드
- 이름 첫 글자 기반 이니셜 아바타 표시
- 이름, 이메일 표시

#### 프로필 수정
- **이름 변경:** 인풋 + 저장 버튼
- 유효성 검사: 빈 값 또는 공백만 입력 불가
- 저장 시 Redux 상태 + localStorage 동시 업데이트

#### 계정 관리
- **로그아웃 버튼:** 클릭 시 Redux 초기화 + localStorage 삭제 후 홈(`/`)으로 이동

---

### 3-5. 찜 목록 페이지 (`/wishlist`)

- Header / Footer 포함 (기존 App 레이아웃)
- 비로그인 사용자도 접근 가능

#### 찜 목록

- 찜한 상품을 카드 형태로 나열한다 (홈 화면과 동일 그리드: 모바일 2열 / 태블릿 3열 / 데스크탑 4열)
- 카드 1개에 표시되는 정보: 상품 이미지, 상품명, 가격
- 카드 우상단 하트 버튼 클릭 시 찜 해제 (목록에서 즉시 제거)
- 카드 클릭 시 해당 상품의 상세 페이지로 이동

#### 액션

- **장바구니 담기:** 각 카드에서 바로 장바구니에 1개 추가
- **전체 삭제:** "전체 삭제" 버튼 클릭 시 찜 목록 전체 비우기

#### 빈 목록 상태

- 찜한 상품이 없을 때 안내 문구와 홈으로 이동 버튼 표시

---

### 3-6. 회원가입 페이지 (`/signup`)

- Header / Footer 없는 풀페이지 레이아웃 (로그인 페이지와 동일)
- **입력 필드:** 이름, 이메일, 비밀번호, 비밀번호 확인
- **유효성 검사 (submit 시):**
  - 빈 필드 → 필드 하단 인라인 에러 표시
  - 이메일 형식 오류 → 인라인 에러 표시
  - 비밀번호 4자 미만 → 인라인 에러 표시
  - 비밀번호 확인 불일치 → 인라인 에러 표시
  - 이미 가입된 이메일 → 상단 에러 메시지 표시
- 가입 성공 시 자동 로그인 후 홈(`/`)으로 이동
- 이미 로그인 상태에서 접근 시 홈으로 리다이렉트
- 하단에 로그인 페이지 링크 제공

## 5. 상태 관리

### Redux Store 구조

```js
{
  cart: {
    items: [
      {
        id: 1,
        name: "사과",
        price: 3000,
        image: "/images/apple.jpg",
        quantity: 2
      }
    ]
  },
  auth: {
    user: { email: "test@fruit.com", name: "과일러버" } | null,
    isLoggedIn: false,
    error: null
  },
  wishlist: {
    items: [
      {
        id: 1,
        name: "사과",
        price: 3000,
        image: "/images/apple.jpg"
      }
    ]
  }
}
```

### cartSlice 액션 목록

| 액션 | 설명 |
|---|---|
| `addToCart(product, quantity)` | 상품 추가. 이미 있으면 수량 합산 |
| `removeFromCart(id)` | 상품 제거 |
| `increaseQuantity(id)` | 수량 +1 |
| `decreaseQuantity(id)` | 수량 -1 (1 미만 불가) |

### wishlistSlice 액션 목록

| 액션 | 설명 |
|---|---|
| `toggleWishlist(product)` | 찜 목록에 없으면 추가, 있으면 제거 |
| `removeFromWishlist(id)` | 특정 상품 제거 |
| `clearWishlist()` | 전체 비우기 |

### authSlice 액션 목록

| 액션 | 설명 |
|---|---|
| `login({ email, password })` | mock 사용자 데이터와 대조 후 인증 처리 |
| `signup({ name, email, password })` | mock 사용자 목록에 추가 후 자동 로그인 |
| `logout()` | 로그인 상태 초기화 |
| `clearError()` | 인증 에러 초기화 |
| `updateProfile({ name })` | 사용자 이름 변경 후 localStorage 동기화 |

## 6. 데이터

### 상품 데이터 (`products.json`)
로컬 JSON 파일로 관리. v1에서는 외부 API 연동 없음.

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | number | 고유 식별자 |
| `name` | string | 상품명 (한글) |
| `englishName` | string | 상품명 (영문) |
| `price` | number | 가격 (원) |
| `description` | string | 상세 설명 |
| `origin` | string | 원산지 |
| `weight` | string | 중량 |
| `stock` | number | 재고 수량 |
| `image` | string | 이미지 경로 |
| `category` | string | `"국내산"` \| `"수입산"` |

### 사용자 데이터 (mock, authSlice 내 관리)
실제 서버 없이 Redux 메모리 내에서만 관리. 새로고침 시 초기화됨.

| 필드 | 타입 | 설명 |
|---|---|---|
| `name` | string | 사용자 이름 |
| `email` | string | 이메일 (로그인 ID) |
| `password` | string | 비밀번호 (mock) |
