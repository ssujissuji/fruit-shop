# PRD — 과일 쇼핑몰 (fruit-shop)

**버전:** 1.0  
**작성일:** 2026-05-13  
**상태:** 구현 완료 기준 문서

---

## 1. 프로젝트 개요

### 목적
신선한 과일을 온라인으로 간편하게 구매할 수 있는 쇼핑몰 웹 애플리케이션을 구축한다.

### 목표 사용자
- 마트 방문 없이 집에서 신선한 과일을 주문하고 싶은 일반 소비자
- 모바일/PC 모두 사용 가능한 환경을 원하는 사용자

### 범위
- 상품 탐색 (홈 화면)
- 상품 상세 조회
- 장바구니 관리
- (v1 제외) 결제, 회원가입/로그인, 주문 내역

---

## 2. 화면 목록

| 화면 | URL | 설명 |
|---|---|---|
| 홈 | `/` | 배너 슬라이더 + 전체 상품 목록 |
| 상품 상세 | `/product/:id` | 과일 상세 정보 + 구매 액션 |
| 장바구니 | `/cart` | 담긴 상품 목록 + 수량 관리 |

---

## 3. 화면별 기능 명세

### 3-1. 홈 화면 (`/`)

#### 배너 슬라이더
- 3개 이상의 프로모션 배너 이미지를 자동 슬라이드한다
- 좌/우 이동 버튼으로 수동 전환이 가능하다
- 현재 슬라이드 위치를 인디케이터(점)로 표시한다
- 자동 슬라이드 간격: 3초

#### 상품 목록
- 전체 과일 상품을 카드 형태로 나열한다
- 카드 1개에 표시되는 정보: 상품 이미지, 상품명, 가격
- 카드 클릭 시 해당 상품의 상세 페이지로 이동한다
- 레이아웃: 모바일 2열 / 태블릿 3열 / 데스크탑 4열 (반응형)

---

### 3-2. 상품 상세 페이지 (`/product/:id`)

#### 표시 정보
| 항목 | 내용 |
|---|---|
| 상품 이미지 | 대표 이미지 1장 |
| 상품명 | 한글명 + 영문명 |
| 가격 | 원화 형식 (예: 3,000원) |
| 원산지 | 텍스트 |
| 중량 | 텍스트 |
| 상품 설명 | 상세 텍스트 |
| 재고 | 재고 수량 표시 |

#### 구매 액션
- **수량 선택:** `-` / `+` 버튼으로 수량을 조절한다
  - 최소 수량: 1 (1 미만으로 감소 불가)
  - 최대 수량: 재고 수량 이하
- **장바구니 담기:** 선택 수량만큼 장바구니에 추가한다
  - 이미 담긴 상품이면 수량을 합산한다
- **구매하기:** 장바구니에 추가 후 장바구니 페이지로 즉시 이동한다
- 존재하지 않는 id 접근 시 홈으로 리다이렉트한다

---

### 3-3. 장바구니 페이지 (`/cart`)

#### 상품 목록
- 담긴 상품별로 행을 구성한다
- 행 표시 항목: 상품 이미지, 상품명, 단가, 수량, 소계 (단가 × 수량)

#### 수량 관리
- 각 상품의 `-` / `+` 버튼으로 수량을 조절한다
  - 최소 수량: 1
- **삭제 버튼:** 해당 상품을 장바구니에서 제거한다

#### 주문 요약
- 전체 합계 금액을 표시한다 (담긴 상품 소계의 합)
- **주문하기 버튼:** v1에서는 UI만 구현 (실제 결제 기능 제외)

#### 빈 장바구니 상태
- 담긴 상품이 없을 때 안내 문구와 홈으로 이동 버튼을 표시한다

## 4. 공통 컴포넌트

### Header
- 로고 (클릭 시 홈 이동)
- 장바구니 아이콘 + 담긴 상품 수량 뱃지

### Footer
- 쇼핑몰 이름, 간단한 안내 문구

---

## 5. 상태 관리

### Redux Store 구조

```js
{
  cart: {
    items: [
      {
        id: 1,
        name: "사과",
        price: 3000,
        image: "/images/apple.jpg",
        quantity: 2
      }
    ]
  }
}
```

### cartSlice 액션 목록

| 액션 | 설명 |
|---|---|
| `addToCart(product, quantity)` | 상품 추가. 이미 있으면 수량 합산 |
| `removeFromCart(id)` | 상품 제거 |
| `increaseQuantity(id)` | 수량 +1 |
| `decreaseQuantity(id)` | 수량 -1 (1 미만 불가) |

---

## 6. 데이터

### 상품 데이터 (`products.json`)
로컬 JSON 파일로 관리. v1에서는 외부 API 연동 없음.

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | number | 고유 식별자 |
| `name` | string | 상품명 (한글) |
| `englishName` | string | 상품명 (영문) |
| `price` | number | 가격 (원) |
| `description` | string | 상세 설명 |
| `origin` | string | 원산지 |
| `weight` | string | 중량 |
| `stock` | number | 재고 수량 |
| `image` | string | 이미지 경로 |
| `category` | string | `"국내산"` \| `"수입산"` |

## 8. 완료 조건 (Acceptance Criteria)

- [ ] 홈 화면에서 배너 자동 슬라이드가 동작한다
- [ ] 상품 카드 클릭 시 올바른 상세 페이지로 이동한다
- [ ] 상세 페이지에서 수량 조절 후 장바구니 담기가 정상 동작한다
- [ ] 이미 담긴 상품을 추가하면 수량이 합산된다
- [ ] 장바구니 페이지에서 수량 변경 및 삭제가 정상 동작한다
- [ ] Header의 장바구니 뱃지가 실시간으로 수량을 반영한다
- [ ] 빈 장바구니 상태에서 안내 UI가 표시된다
- [ ] 모바일 / 데스크탑 레이아웃이 반응형으로 동작한다
